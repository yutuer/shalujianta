# 手牌预览功能使用说明（最终版）

## 概述

已为 BattleScene 添加手牌预览功能，您可以在编辑器中实时预览和调试手牌的扇形展开效果。

## ⚠️ 重要更新（2026-03-24 第四次更新）

**Godot 调试模式自动检测**：
1. ✅ **自动检测调试模式**：使用 Godot 内置 API 自动判断
2. ✅ **仅调试模式隐藏正式手牌**：调试/编辑器模式下正式手牌自动隐藏
3. ✅ **移除手动开关**：不再需要手动设置 `debugModeOnly` 字段
4. ✅ **智能切换**：运行正式游戏时自动显示正式手牌

## 自动调试模式检测

现在使用 **Godot 内置的调试检测方法**：

```csharp
private bool IsDebugMode()
{
    bool isDebug = OS.IsDebugBuild();      // 判断是否为调试构建
    bool isEditor = Engine.IsEditorHint();  // 判断是否在编辑器中运行

    GD.Print($"[HandCardPreview] 调试检测: IsDebugBuild={isDebug}, IsEditorHint={isEditor}");

    return isDebug || isEditor;
}
```

### 检测逻辑

| 运行场景 | `IsDebugBuild` | `IsEditorHint` | 效果 |
|---------|---------------|----------------|------|
| **编辑器中运行（F5）** | true | true | 隐藏正式手牌，显示预览卡牌 |
| **调试构建运行** | true | false | 隐藏正式手牌，显示预览卡牌 |
| **发布构建运行** | false | false | 显示正式手牌，隐藏预览卡牌 |

1. 打开 Godot 编辑器
2. 打开 `BattleScene.tscn` 场景

   - ✅ 只显示**预览卡牌**（5张）

## 自动模式切换

### 调试模式（编辑器/调试构建）

当检测到调试模式时：
- `IsDebugMode()` 返回 `true`
- 自动隐藏 HandArea 正式手牌区域
- 显示预览卡牌和调试面板
- 控制台输出：`[HandCardPreview] 检测到调试模式，隐藏正式手牌`

### 发布模式（发布构建）

当检测到发布模式时：
- `IsDebugMode()` 返回 `false`
- 显示 HandArea 正式手牌区域
- 隐藏预览卡牌（但不销毁）
- 控制台输出：无（静默模式）

## 可调参数说明

| 参数 | 范围 | 说明 | 建议值 |
|------|------|------|--------|
| **手牌数量** | 1-10 | 调整预览的手牌数量 | 5-7 |
| **扇形角度** | 0-30° | 控制悬停时相邻卡牌的展开角度 | 10-15° |
| **远端角度** | 0-45° | 控制离悬停卡牌较远的卡牌的最大角度 | 20-30° |
| **基础Y偏移** | 0-60 | 控制远端卡牌的Y轴偏移量 | 20-40 |
| **抬升高度** | 0-150 | 控制悬停卡牌的Y轴抬升高度 | 60-100 |
| **重叠间距** | 50-150 | 控制卡牌之间的水平间距 | 90-120 |
| **悬停间距** | 80-180 | 悬停时卡牌之间的展开间距 | 110-140 |

## 扇形展开算法说明

### 悬停状态
当鼠标悬停在某张手牌上时：
1. **悬停卡牌**: Y轴抬升 `hoverLift`，保持水平（rotation=0）
2. **相邻卡牌 (±1-2)**: 根据距离计算扇形角度，Y轴轻微抬升
3. **远端卡牌 (±3+)**: 使用远端角度旋转，Y轴向下偏移

### 正常状态
当没有悬停时：
- 所有卡牌以中心为基准，形成轻微的扇形排列
- 角度和偏移量都按比例缩小

## 与正式系统的集成

当前预览组件是**独立的**，仅用于调试目的。正式的手牌系统逻辑位于：

- **CardLayoutManager.cs** - `Scripts/Battle/CardLayoutManager.cs`
  - 管理实际战斗中的手牌布局
  - 与 BattleManager 集成
  - 支持拖拽、使用卡牌等交互

预览组件的参数默认值与 CardLayoutManager 中的常量保持一致，便于对照调试。

## 导出参数到正式系统

调试完成后，可以将满意的参数值复制到 CardLayoutManager.cs 中：

## 技术细节

- **自动调试检测**：使用 `OS.IsDebugBuild()` 和 `Engine.IsEditorHint()`
- **智能模式切换**：根据运行环境自动切换调试/发布模式
- **独立 CanvasLayer**：调试面板使用独立的 CanvasLayer（Layer=100）
- **预览卡牌控制**：调试模式下显示，发布模式下隐藏
- **ZIndex 管理**：
  - 调试面板: ZIndex = 9999
  - 预览卡牌: ZIndex = 100

## Godot 调试 API 详解

### OS.IsDebugBuild()
- **作用**：判断当前是否为调试构建
- **返回值**：`true` = 调试构建，`false` = 发布构建
- **使用场景**：区分调试和发布版本

### Engine.IsEditorHint()
- **作用**：判断当前是否在编辑器中运行
- **返回值**：`true` = 编辑器中，`false` = 独立运行
- **使用场景**：区分编辑器环境和独立运行

### 组合使用
```csharp
// 推荐用法：任一条件满足即为调试模式
bool isDebug = OS.IsDebugBuild() || Engine.IsEditorHint();

// 仅编辑器模式
bool isEditorOnly = Engine.IsEditorHint();

// 仅调试构建（不包括编辑器）
bool isDebugBuildOnly = OS.IsDebugBuild() && !Engine.IsEditorHint();
```

## 常见问题

**Q: 如何切换到正常游戏模式？**
A: 有两种方法：
1. 从 BattleScene.tscn 中**移除** HandCardPreview 节点
2. 或者构建**发布版本**（Release Build）

**Q: 调试面板挡住了视线怎么办？**
A: 点击"切换调试面板"按钮可以隐藏/显示面板。面板已移至右上角，通常不会遮挡主要游戏区域。

**Q: 如何调整调试面板的位置？**
A: 目前调试面板固定在右上角。如果需要调整，可以修改 `PositionDebugPanel()` 方法中的参数。

**Q: 预览效果和实际效果不一样？**
A: 预览组件使用独立参数，需要手动同步到 CardLayoutManager。

**Q: 如何确认当前是否在调试模式？**
A: 查看控制台输出，会显示：
- `[HandCardPreview] 调试检测: IsDebugBuild=X, IsEditorHint=Y`
- `[HandCardPreview] 检测到调试模式，隐藏正式手牌`（仅调试模式下）

## 调试建议

### 新手调试流程

1. **启动游戏**：按 F5 在编辑器中运行
2. **观察控制台**：确认调试模式检测成功
3. **调整参数**：拖动滑块实时调整扇形参数
4. **测试交互**：鼠标悬停到卡牌上观察效果
5. **记录参数**：记下满意的参数值
6. **导出参数**：将参数复制到 CardLayoutManager.cs

### 高级调试技巧

**分层调试**：
- 1-3张卡牌：调试轻微的扇形效果
- 5-7张卡牌：调试中等密度的扇形效果
- 8-10张卡牌：调试高密度的手牌展开

**分离参数调试**：
- 第一轮：只调间距参数（overlapOffset, hoverSpreadOffset）
- 第二轮：只调角度参数（fanSpreadAngle, farFanAngle）
- 第三轮：只调高度参数（hoverLift, fanBaseYOffset）
- 第四轮：综合微调所有参数
