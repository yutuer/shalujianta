我准备做使用godot做一款类似杀戮尖塔的游戏, 要求语言使用c#.  场景, ui均使用godot来构建, 能看出大概的布局即可. 我会同时附加一张游戏规则的图片(见  ./游玩效果图.png)

# 项目介绍

	该项目是一款回合制卡牌游戏.  玩家每回合会拥有一定数量的卡牌, 每个卡牌有自己的作用. 
	玩家通过出卡牌, 来与敌人进行战斗, 直到有玩家的血量为0, 或者玩家主动结束游戏. 当玩家胜利的时候, 会进入下一关卡, 重新开始.
	
# 游戏介绍
	
	游戏由玩家, 卡牌, 敌人组成.
	1. 玩家: 玩家是游戏的主要角色, 有血量, 攻击力, 防御力, 能量(算力), 回合抽卡数量, 最大手牌数量, 当前手牌数量 等属性.
		1. 玩家有职业的, 这是为了区分不同种类的玩家有不同的默认卡牌组合. 目前就先设置一种职业
	2. 卡牌: 最关键的概念就是卡牌. 卡牌是游戏的最主要元素, 有攻击, 恢复血量, 增加护盾等作用. 卡牌有算力消费的属性, 表示打出这张卡牌(手牌)需要消费玩家多少算力. 
	   每一张卡牌的属性都是事先定制好的, 以后可以通过增加卡牌来扩展游戏的功能, 也可以通过修改卡牌的属性来改变游戏.
	3. 卡牌组合: 每次进入新的一局游戏会预先生成一批卡牌, 这批卡牌组成了这局游戏的卡牌组合.
		1. 不同的职业会有不同的卡牌组合
		2. 以后也会有某些条件往这局卡牌组合里面增减卡牌
	4. 抽卡池: 抽卡池是指初始化时从卡牌组合复制过来的一组卡牌. 这样玩家每个回合可以从抽卡池里面按照顺序抽一定数量的卡.
	5. 手牌: 手牌是指卡牌从抽卡池进入到玩家的手中, 玩家可以操作的卡牌. 此时的手牌除了原来卡牌上面自带的属性外还会增加上其他的属性
		1. 控制手牌丢弃的属性: 该手牌每回合结束后不再会被丢弃.
		2. 每个回合消费算力减少的属性: 该手牌每回合结束后会减少消费的算力  
	敌人: 敌人是游戏的敌人, 有血量, 攻击力, 防御力等属性. 
	回合数: 当前进行到哪一个回合了. 玩家和敌人都行动完毕叫一回合, 有的游戏局有回合数限制. 每一个回合结束后, 玩家的手牌会全部被丢弃掉(除非拥有不会被丢弃的属性). 


# 游戏规则

	1. 当新的一局游戏开始的时候,  进入战斗场景ui(示意图见 ./游玩效果图.png). 卡牌组合会初始化, 抽卡池会清空, 然后将卡牌组合复制一份并打乱顺序放入抽卡池中.  玩家的手牌会显示在画面下方中央, 整个手牌呈现扇形打开.  
	2. 每个回合开始的时候,  会根据玩家的抽卡数这个属性来从抽卡池中抽取相同数量的卡牌放到玩家的手中, 同时将抽取后的卡牌从抽卡池中移除. 抽卡池中的卡牌顺序是随机的, 玩家不能预测到下一张卡牌是什么. 当抽卡池中的卡牌不够给玩家手牌的时候, 会重新将卡牌组合*1的卡牌放入到抽卡池中供玩家抽取.
	3. 每个回合开始的时候, 会将敌人本回合执行指令后造成的结果告诉玩家(比如本回合敌人攻击会造成玩家损失多少血量, 敌人给自己加盾会多出多少护盾, 回血多少等等). 
	4. 每个回合玩家先出牌, 直到主动选择结束当前回合(可以是因为算力不够, 也可以是因为单纯的想结束本回合). 然后敌人开始根据回合预设开始行动.
	5. 当玩家的血量为0的时候, 此局游戏失败, 可以选择重新开始或者退出此局游戏. 当所有敌人的血量变成0的时候, 此局游戏胜利.
